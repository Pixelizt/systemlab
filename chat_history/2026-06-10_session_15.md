# Session 15 — 2026-06-10

## Issues Fixed

### 1. Real-time task sync (cross-tab)
**Bug:** `onValueCallback` used `remoteS._rev <= S._rev` to suppress echo. The `<=` meant if two tabs had the same `_rev` (same millisecond persist), the incoming update was silently dropped.  
**Fix:** Changed to `remoteS._rev < S._rev`. Equal revisions now pass through — the `isSyncing` flag prevents any persist loop.

### 2. LeadGen data not saving to cloud in real-time
**Root cause A:** Same `_rev` issue above — cross-tab updates were dropped.  
**Root cause B:** LeadGen inputs use `onchange` (fires on blur). If a user typed a value and then switched browser tabs without clicking elsewhere, the `onchange` never fired and `persist()` was never called.  
**Fix:** Added `persist()` call in the existing `visibilitychange` listener when `document.visibilityState === 'hidden'`. This flushes any uncommitted field values to Firebase whenever the user switches tabs or minimises the window.

### 3. lgRender sync on incoming remote update
**Fix:** Removed the `v-leadgen view-active` guard from `lgRender()` inside `onValueCallback`. LeadGen state now always updates in-memory when a remote sync fires; `showView('leadgen')` already gates the actual DOM render when the user navigates there.
