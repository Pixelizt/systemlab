# Session 15 — 2026-06-10

## Issues Fixed

### 1. Real-time task sync (cross-tab)
**Bug:** `onValueCallback` used `remoteS._rev <= S._rev` to suppress echo. The `<=` meant if two tabs had the same `_rev` (same millisecond persist), the incoming update was silently dropped.  
**Fix:** Changed to `remoteS._rev < S._rev`. Equal revisions now pass through — the `isSyncing` flag prevents any persist loop.

### 2. LeadGen data not saving to cloud in real-time
**Root cause A:** Same `_rev` issue above — cross-tab updates were dropped.  
**Root cause B:** LeadGen inputs use `onchange` (fires on blur). If a user typed a value and then switched browser tabs without clicking elsewhere, the `onchange` never fired and `persist()` was never called.  
**Fix:** Added `persist()` call in the existing `visibilitychange` listener when `document.visibilityState === 'hidden'`. This flushes any uncommitted field values to Firebase whenever the user switches tabs or minimises the window.

### 3. lgRender sync on incoming remote update
**Fix:** Removed the `v-leadgen view-active` guard from `lgRender()` inside `onValueCallback`. LeadGen state now always updates in-memory when a remote sync fires; `showView('leadgen')` already gates the actual DOM render when the user navigates there.

## Document Vault + Prompt Vault Redesign

### Document Vault
- **Quick Access strip**: Pin any link → it appears as a chip at the top of the vault for instant one-click open. Unpin via ✕ on hover.
- **Smart type-filter chips**: URL auto-detection (Google Workspace, Notion, Design/Figma, PM Tools, Video, etc.). One-click filter by type.
- **Client/Project tag**: New field on every link. Shows as a blue chip on the card. Included in search.
- **Grid/List view toggle**: ▦ Grid (default) and ☰ List for dense collections.
- **Data schema**: Added `pinned: boolean`, `client: string` fields (backward compatible).

### Prompt Vault
- **{{variable}} interpolation**: Any `{{variable_name}}` in a prompt text becomes an auto-detected fill-in field. Click "⚡ Use" → modal opens with one input per variable → live preview updates as you type → "Copy Prompt" copies the complete filled prompt.
- **Prompts with no variables**: "📋 Copy" button copies directly (no modal).
- **Recently Used strip**: Last 5 used prompts shown as quick-launch chips (stored in localStorage).
- **Tab-style categories**: Replaces pill filters. All | ⭐ Saved | [categories] with count badges.
- **Renamed**: "Prompt Library" → "Prompt Vault" for agency branding.
- **Usage counter**: `usageCount` increments on every use, displayed on card.
