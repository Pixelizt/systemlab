## Bugs Fixed

### 1. Recurring task respawning after deletion (two-part fix)

**Root cause:** Firebase RTDB silently drops empty arrays — when `S.recurring = []` is persisted, Firebase stores `null`. On reload, `safeMerge(null, localRecurring, [])` falls back to the stale localStorage value, restoring all deleted recurring tasks.

**Fix A — `onValueCallback`** (already in session 15 handoff):
```js
S.recurring = Array.isArray(remoteS.recurring) ? remoteS.recurring : [];
```

**Fix B — `connectToWorkspace` initial load** (line ~1779, applied this session):
Same pattern. The `connectToWorkspace` function had its own `safeMerge(remoteS.recurring, S.recurring, [])` call that was not covered by Fix A. Changed to:
```js
S.recurring = Array.isArray(remoteS.recurring) ? remoteS.recurring : [];
```

### 2. Task edits lost on window close

**Fix:** Added `beforeunload` handler:
```js
window.addEventListener('beforeunload', () => { try { persist(); } catch(e) {} });
```

### 3. Manual attendance time entry

**Added:** Time input fields (`loginTime`, `logoutTime`) in the attendance modal. Uses `to24h`/`from24h`/`saveAttTime` helpers — stores in 24h format internally, displays in 12h AM/PM format.

### 4. deleteEmployee not removing tasks

**Root cause:** `S.tasks[curEmp]` treated the tasks array as an object keyed by empId — always `undefined`, so tasks were never deleted.

**Fix:**
```js
if (S.tasks) S.tasks = S.tasks.filter(t => t.empId !== curEmp);
```

### 5. `injectSystemSOPs` injecting `status: 'published'`

**Root cause:** The app only uses `'draft'`, `'active'`, `'review'`. The injected Team Daily SOP had `status: 'published'`, which meant the dashboard's "Active" counter never included it (filters for `=== 'active'`).

**Fix:** Added migration on inject + changed injection default:
```js
const existing = S.sops.find(s => s.id === SOP_TEAM_DAILY_ID);
if (existing) {
  if (existing.status === 'published') { existing.status = 'active'; persist(); }
  return;
}
// ...injection now uses status: 'active'
```

### 6. PDF export issues

- **"HTML BLOCK" label bleeds to new page:** Hidden in print with `.html-block-label { display: none !important; }`
- **Missing print colors:** Injected `print-color-adjust: exact` CSS into every iframe's `srcdoc` via `htmlBlockRender`
- **Steps splitting across page breaks:** Added `break-inside: avoid; page-break-inside: avoid` to `.step`, `.hb`, `.rb`, `.cls`, `.rr`, `.html-block-wrap`

### 7. Null reference crash on dashboard render

**Bug:** `renderDashboard` called `document.getElementById('dash-activity').innerHTML` but no element with id `dash-activity` existed in the HTML — threw `TypeError` on every render.

**Fix:** Removed the `dash-activity` element entirely, replaced with live team snapshot (see Dashboard Rebuild below).

## Architecture Reminders

- `safeMerge(remote, local, fallback)` — safe merge that protects against Firebase returning null for previously-populated arrays. **Use for tasks, att, depts, sops, prompts, docs.** Do NOT use for `recurring` — use `Array.isArray(remote) ? remote : []` instead.
- `isSyncing` flag — `persist()` returns early during Firebase sync to avoid write loops
- `_recurringSpawnedDate` — prevents double-spawn of recurring tasks
- SOP status values: `'draft'`, `'active'`, `'review'` — never `'published'`
- Firebase RTDB drops empty arrays silently — stores `null` instead of `[]`
